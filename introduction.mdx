## What is hawkx?

hawkx is a CLI tool that lets you fetch tweets, bookmarks, and post threads from X (formerly Twitter) using your existing Chrome browser session. No API key required, no OAuth flow — it simply uses the cookies from your logged-in Chrome profile.

## Why hawkx?

X's API is paywalled, making it difficult and expensive to access basic data programmatically. hawkx solves this by acting as a thin wrapper over the same requests the X web app makes. If you can view it in your browser, you can fetch it with hawkx.

## What you can do

- Fetch recent tweets from any user
- Get your bookmarks
- Retrieve individual posts with or without replies
- Extract image URLs from tweets
- Switch between Chrome profiles
- Output as JSON or plain text

## Requirements

<Warning>
  hawkx currently only works on macOS due to Chrome cookie decryption using Keychain.
</Warning>

Before you can use hawkx, you need:

- **macOS** — Chrome cookie decryption relies on macOS Keychain
- **Python 3.10 or later** — Required for the CLI tool
- **Chrome browser** — Must have Chrome installed
- **Logged into X** — You must be signed into x.com in Chrome

## Output format

By default, hawkx outputs JSON with the following fields:

- `id` — Tweet ID
- `text` — Full text of the tweet
- `created_at` — Timestamp
- `user` — Screen name of the author
- `favorites` — Like count
- `retweets` — Retweet count
- `replies` — Reply count

Use the `-I` flag to include an `images` array, and `-R` for reply metadata.
