## Get up and running

This guide will walk you through installing hawkx and running your first commands.

<Steps>
  <Step title="Install hawkx">
    Install hawkx using pipx (recommended) or pip:
    
    <CodeGroup>
    ```bash pipx
    pipx install hawkx
    ```
    
    ```bash pip
    pip install --user hawkx
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Verify Chrome login">
    Make sure you're logged into [x.com](https://x.com) in Chrome. hawkx uses your browser's session cookies to authenticate.
    
    <Info>
      If you use a Chrome profile other than "Profile 1", set it with: `hawkx setprofile "Your Profile Name"`
    </Info>
  </Step>
  
  <Step title="Run your first command">
    Fetch recent tweets from a user:
    
    ```bash
    hawkx getuser elonmusk 10
    ```
    
    This fetches the 10 most recent tweets from @elonmusk.
    
    **Expected output:**
    
    ```json
    [
      {
        "id": "1234567890",
        "text": "Tweet text here...",
        "created_at": "Wed Mar 04 12:34:56 +0000 2026",
        "user": "elonmusk",
        "favorites": 15234,
        "retweets": 3421,
        "replies": 892
      },
      // ... 9 more tweets
    ]
    ```
  </Step>
  
  <Step title="Try other commands">
    Explore what else you can do:
    
    **Fetch tweets with image URLs:**
    ```bash
    hawkx getuser elonmusk 20 -I
    ```
    
    **Get text-only output:**
    ```bash
    hawkx getuser elonmusk 5 -t
    ```
    
    **Fetch your bookmarks:**
    ```bash
    hawkx getbookmarks 50
    ```
    
    **Get a specific tweet with replies:**
    ```bash
    hawkx getpost 1234567890 -R
    ```
  </Step>
</Steps>

## Common use cases

### Fetch tweets from a user

You can use either the username or a full URL:

<CodeGroup>
```bash Username
hawkx getuser elonmusk 20
```

```bash URL
hawkx getuser https://x.com/elonmusk 20
```
</CodeGroup>

Both commands fetch the same data.

### Include images

Add the `-I` flag to get image URLs in the output:

```bash
hawkx getuser elonmusk 20 -I
```

The output will include an `images` array:

```json
{
  "id": "1234567890",
  "text": "Check out this image!",
  "images": [
    "https://pbs.twimg.com/media/abc123.jpg",
    "https://pbs.twimg.com/media/def456.jpg"
  ],
  // ... other fields
}
```

### Get your bookmarks

Fetch tweets you've bookmarked:

```bash
hawkx getbookmarks 100
```

<Tip>
  You can combine flags: `hawkx getbookmarks 100 -I -t` gives you text-only output with images.
</Tip>

### Fetch a specific tweet

Get a single tweet by ID or URL:

<CodeGroup>
```bash Tweet ID
hawkx getpost 1234567890
```

```bash Tweet URL
hawkx getpost https://x.com/elonmusk/status/1234567890
```
</CodeGroup>

Add `-R` to include replies:

```bash
hawkx getpost 1234567890 -R
```

## Available flags

All fetch commands support these flags:

| Flag | Description |
|------|-------------|
| `-R` | Include replies or reply metadata |
| `-I` | Include image URLs in the output |
| `-t` | Text-only output (prints just tweet text, no JSON) |

## Understanding the output

By default, hawkx outputs JSON with these fields:

- `id` — Unique tweet identifier
- `text` — Full text content
- `created_at` — Timestamp in X's format
- `user` — Screen name (username)
- `favorites` — Number of likes
- `retweets` — Number of retweets
- `replies` — Number of replies

With `-I`, you get:
- `images` — Array of image URLs

With `-R` on replies, you get:
- `in_reply_to` — Object with `status_id` and `screen_name`
