## Overview

The `getuser` command retrieves recent tweets from a specified user's timeline. It accepts a screen name or full X/Twitter URL and returns structured JSON data containing tweet information.

## Syntax

```bash
hawkx getuser <user> [limit] [options]
```

## Parameters

<ParamField path="user" type="string" required>
  Screen name or URL of the user to fetch tweets from. Accepts:
  - Screen name: `elonmusk`
  - With @ symbol: `@elonmusk`
  - Full URL: `https://x.com/elonmusk`
</ParamField>

<ParamField path="limit" type="integer" default="40">
  Number of posts to retrieve. Defaults to 40 if not specified.
</ParamField>

## Options

| Flag | Description |
|------|-------------|
| `-R`, `--replies` | Include replies and reply metadata in the output |
| `-I`, `--images` | Include image URLs in the output |
| `-t`, `--text` | Print only tweet text (one per line, no JSON) |

## Usage Examples

### Basic Usage

Fetch 40 tweets from a user (default):

```bash
hawkx getuser elonmusk
```

### Fetch Specific Number of Tweets

Get 20 tweets from a user:

```bash
hawkx getuser elonmusk 20
```

### Using Full URL

You can also use the full X/Twitter URL:

```bash
hawkx getuser https://x.com/MohiniWealth 100
```

### Include Images

Fetch tweets with image URLs:

```bash
hawkx getuser elonmusk 20 -I
```

### Include Replies

Get tweets including replies and reply metadata:

```bash
hawkx getuser elonmusk 50 -R
```

### Text-Only Output

Print only the text content without JSON formatting:

```bash
hawkx getuser MohiniWealth 100 -t
```

### Combine Multiple Options

Fetch tweets with both images and reply metadata:

```bash
hawkx getuser elonmusk 30 -R -I
```

## Output Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique tweet identifier |
| `text` | string | Full text content of the tweet |
| `created_at` | string | Timestamp when the tweet was posted |
| `user` | string | Screen name of the tweet author |
| `favorites` | integer | Number of likes/favorites |
| `retweets` | integer | Number of retweets |
| `replies` | integer | Number of replies |
| `images` | array | Image URLs (only with `-I` flag) |
| `in_reply_to` | object | Reply metadata (only with `-R` flag for reply tweets) |

## Common Use Cases

### Research and Analysis

Collect tweets for sentiment analysis or research:

```bash
hawkx getuser researcher_account 200 > tweets.json
```

### Archive User Timeline

Save a user's recent tweets with all metadata:

```bash
hawkx getuser username 500 -I -R > archive.json
```

### Quick Text Extraction

Extract just the text for quick reading or processing:

```bash
hawkx getuser username 50 -t | grep "keyword"
```

## Notes

<Note>
The user parameter is flexible and accepts screen names with or without the `@` symbol, as well as full X/Twitter URLs.
</Note>

<Warning>
By default, replies are excluded from the timeline. Use the `-R` flag to include them.
</Warning>

<Tip>
Combine the `-t` flag with command-line tools like `grep`, `wc`, or `less` for quick text processing without dealing with JSON.
</Tip>
